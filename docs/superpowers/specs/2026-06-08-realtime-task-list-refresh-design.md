# 实时进度任务列表刷新与展示重构设计

## 背景

实时任务页当前由 `frontend/src/pages/Home/TaskList.tsx` 同时负责当前任务汇总轮询、Tab 明细轮询、历史任务轮询、请求去重、列表合并、分页处理和页面展示。多个刷新路径共享同一组件状态，导致逻辑难读，也容易在任务切换、Tab 切换、分页切换或请求乱序时出现旧数据残留、列表闪烁和展示状态不一致。

项目里已经有 `frontend/src/pages/Home/taskRows.ts` 承担部分任务行合并和历史过滤逻辑。本次重构沿用这个方向，把实时进度页的数据刷新与展示拆开，并保留当前接口和视觉风格。

## 目标

- 让实时任务汇总刷新、Tab 明细刷新、历史任务刷新职责清晰分离。
- 避免过期请求覆盖新状态。
- 切换任务、Tab、分页时清空和保留列表的规则稳定可预测。
- 保持当前 Ant Design 操作台风格，优化实时明细空态、分页、排序和文件行展示。
- 增加单元测试覆盖刷新策略和数据归一化逻辑。

## 非目标

- 不改后端 `/job?current=1` 接口契约。
- 不改历史任务筛选语义。
- 不重做实时进度页视觉设计。
- 不引入新的全局状态管理库。

## 推荐方案

采用“数据层 hook + 展示组件 + 纯工具函数”的拆分方式。

### 数据层

新增或调整本地模块，保持在 `frontend/src/pages/Home/` 目录内：

- `useRealtimeTask(jobId, enabled)`：负责当前任务汇总轮询、速度和剩余时间计算、当前任务身份识别。
- `useRealtimeTaskItems(jobId, currentTask, activeTab, page)`：负责实时 Tab 明细列表刷新、分页总数、加载态和请求乱序保护。
- `taskRows.ts`：继续承载纯函数，包括接口返回归一化、任务行 key、稳定合并、排序、分页派生和历史过滤。

`TaskList.tsx` 保留为页面编排组件，负责历史任务列表、实时卡片组合和动作按钮。实时文件行、指标区和 Tab 内容拆成小展示组件，避免刷新逻辑嵌入 JSX 深层结构。

### 刷新规则

- 当前任务汇总在实时视图启用时立即拉取一次，并每 3 秒轮询。
- 历史任务只在历史视图启用时拉取和轮询。
- `运行中` Tab 使用当前任务汇总里的 `doingTask`，不额外请求后端明细接口。
- `等待`、`成功`、`失败`、`其他` Tab 使用 `/job?current=1&status=...&pageSize=...&pageNum=...` 获取分页明细。
- 任务身份变化时清空实时 Tab 明细，并重置 Tab 分页到第一页。
- Tab 变化时清空旧 Tab 明细，并重置分页到第一页。
- 分页变化时请求新页数据，不把旧页数据混入新页；请求期间可显示加载态。
- 同一页轮询刷新时使用稳定合并，未变化行保留引用，降低不必要渲染和视觉跳动。
- 所有实时请求使用单调递增 request id。响应返回时只有仍匹配当前请求序号、当前任务、当前 Tab 和当前页时才落入状态。

### 展示规则

- 当前无实时任务时显示“当前没有正在同步的任务”空态。
- 当前任务存在但 Tab 无明细时显示“暂无{Tab 名称}任务”空态。
- Tab 数量来自当前汇总的 `num` 字段。
- `运行中` Tab 总数来自 `doingTask.length`，其他 Tab 总数来自分页接口 `count`。
- 任务明细排序统一在数据层完成，按 `createTime` 降序，`id` 作为稳定兜底。
- 文件行保持现有字段：操作类型、文件名、来源目录、目标目录、文件大小、进度或状态信息。
- 失败 Tab 优先展示错误提示；非运行中且无错误信息时展示创建时间。

## 错误处理

- 当前任务汇总请求短暂失败时保留最后一次可见快照，避免实时卡片闪烁为空。
- Tab 明细请求失败时，仅当这是当前有效请求且是显式切换或分页加载时，才清空明细并显示空态。
- 旧请求返回后不更新加载态，不覆盖列表。
- 任务动作成功后主动刷新历史列表和当前任务汇总。

## 测试计划

新增或扩展前端测试：

- 接口返回归一化：`CurrentTaskData`、分页 `PageData<TaskItem>`、数组和 `null` 的处理。
- 明细列表合并：同一页轮询保留未变化行引用，变化行替换。
- 分页策略：分页变化时使用新页数据，不混入上一页数据。
- 任务切换策略：当前任务身份变化时清空旧明细。
- Tab 策略：`运行中` Tab 不请求后端明细，其他 Tab 使用分页接口总数。
- 历史过滤：保留现有当前任务隐藏和运行中记录过滤行为。

## 验收标准

- `frontend/src/pages/Home/TaskList.tsx` 中实时明细刷新逻辑不再直接堆叠多个 request ref 和分页判断。
- 实时进度页在任务切换、Tab 切换、分页切换和轮询刷新时不展示明显旧数据残留。
- 当前任务汇总、实时明细和历史任务可以独立启停轮询。
- 前端测试通过。
- 页面仍保留当前实时卡片、指标、Tab、文件行和历史任务表格的操作台风格。
